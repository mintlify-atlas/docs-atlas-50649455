---
title: "unc list-architectures"
description: "List supported model architectures"
---

The `list-architectures` command displays all model architectures currently supported by UNC for compilation.

## Usage

```bash
unc list-architectures
```

## Parameters

<ParamField path="--verbose" type="boolean">
  Enable verbose debug logging (global flag).
</ParamField>

## Example

```bash
unc list-architectures
```

Output:
```
Supported architectures:
  LlamaForCausalLM               LLaMA, LLaMA-2, LLaMA-3
  MistralForCausalLM             Mistral, Mixtral
  Qwen2ForCausalLM               Qwen, Qwen-2
  PhiForCausalLM                 Phi-2, Phi-3
  GemmaForCausalLM               Gemma, Gemma-2
```

## Supported Architectures

### LlamaForCausalLM

Supports the LLaMA family of models:

- **LLaMA** - Original Meta LLaMA models (7B, 13B, 33B, 65B)
- **LLaMA-2** - Meta's improved LLaMA models (7B, 13B, 70B)
- **LLaMA-3** - Latest generation LLaMA models

Example models:
- `meta-llama/Llama-2-7b-hf`
- `meta-llama/Llama-2-13b-chat-hf`
- `meta-llama/Llama-3-8b`

### MistralForCausalLM

Supports Mistral AI models:

- **Mistral** - Mistral base models (7B)
- **Mixtral** - Mixture-of-Experts models (8x7B)

Example models:
- `mistralai/Mistral-7B-v0.1`
- `mistralai/Mistral-7B-Instruct-v0.2`
- `mistralai/Mixtral-8x7B-v0.1`

### Qwen2ForCausalLM

Supports Alibaba Qwen models:

- **Qwen** - Original Qwen models
- **Qwen-2** - Second generation models

Example models:
- `Qwen/Qwen2-7B`
- `Qwen/Qwen2-7B-Instruct`
- `Qwen/Qwen1.5-14B`

### PhiForCausalLM

Supports Microsoft Phi small language models:

- **Phi-2** - 2.7B parameter model
- **Phi-3** - Latest generation Phi models

Example models:
- `microsoft/phi-2`
- `microsoft/phi-3-mini-4k-instruct`
- `microsoft/phi-3-small-8k-instruct`

### GemmaForCausalLM

Supports Google Gemma models:

- **Gemma** - Original Gemma models (2B, 7B)
- **Gemma-2** - Second generation models

Example models:
- `google/gemma-2b`
- `google/gemma-7b`
- `google/gemma-2-9b`

## Architecture Detection

UNC automatically detects the model architecture from the HuggingFace `config.json` file when compiling. The architecture is specified in the `architectures` field:

```json
{
  "architectures": ["LlamaForCausalLM"],
  "model_type": "llama",
  ...
}
```

If you attempt to compile an unsupported architecture, you'll see an error during compilation.

## Future Architectures

The following architectures are planned but not yet supported:

- **GPTNeoX** - EleutherAI GPT-NeoX models (currently returns error during compilation)

## Notes

- This command requires no arguments and no model files
- Architecture support is independent of model size
- All architectures support the full range of quantization options (f32, f16, bf16, q8_0, q4_0)
- New architectures are added in UNC releases - check the changelog for updates
