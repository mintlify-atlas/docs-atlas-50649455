# Energy Efficiency

UNC achieves exceptional energy efficiency through compiled Metal kernels that eliminate Python runtime overhead. This guide presents real power measurements and techniques for optimizing energy usage.

## Executive Summary

**UNC vs mlx-lm (TinyLlama 1.1B Q4_0, M1 Pro):**

- **1.35x faster** (152 vs 113 tok/s)
- **25% less GPU power** (11.3W vs 14.1W)
- **1.7x better energy efficiency** (74 vs 125 mJ per token)
- **8.4x fewer CPU instructions** (5.3B vs 31.4B for 200 tokens)

<Tip>
  UNC delivers more performance per watt than any other inference framework on Apple Silicon. This translates to longer battery life, less heat, and sustained performance.
</Tip>

## Power Measurement Methodology

**Hardware**: Apple M1 Pro (14 GPU cores, 16GB, 200 GB/s)  
**Date**: 2026-03-13  
**Tools**: macmon (sudoless Apple Silicon power monitoring, 200ms sampling), /usr/bin/time -l  
**Methodology**: 
- 200 tokens generated per run
- Prompt: "The history of"
- 15-30s cooldown between A/B runs
- Decode phase identified by GPU utilization > 50%

## Energy Per Token

Energy efficiency is measured in millijoules (mJ) per token:

```
Energy/token = Power (W) × Time per token (s) × 1000
```

UNC eliminates Python runtime and framework dispatch overhead:

|--------|---------|---------|------------|
| User CPU time (200 tok) | 0.30s | 0.16s | 3.27s |
| System CPU time | 0.60s | 0.69s | 0.80s |

### Why Fewer CPU Instructions Matter

1. **Less CPU power**: Fewer instructions = lower CPU energy
2. **Less heat**: CPU generates less waste heat
3. **More GPU headroom**: Less thermal impact on GPU throttling
4. **Better battery life**: CPU power savings compound with GPU savings

**UNC AOT executes 8.4x fewer instructions** than mlx-lm (3.75B vs 31.4B).

## Memory Efficiency

<Warning>
  UNC trades memory for compilation speed - it loads BF16 weights and quantizes at init, causing higher peak memory usage.
</Warning>

| Metric | UNC JIT | UNC AOT | mlx-lm Q4 |
|--------|---------|---------|------------|
| **Peak RSS** | 4.24 GB | 4.11 GB | 0.87 GB |
| Peak footprint | 3.61 GB | 3.70 GB | 0.81 GB |
| Q4_0 weights (GPU) | 618 MB | 618 MB | ~618 MB |

mlx-lm's 4-5x lower memory usage comes from loading pre-quantized weights directly.

**Implications**:
- UNC requires 16GB+ RAM for 1-4B models
- mlx-lm can run on 8GB Macs
- Runtime GPU memory identical (both use 618 MB for weights)

## Thermal Behavior

| Metric | UNC | mlx-lm |
|--------|-----|--------|
| GPU temp (idle) | 66-70°C | — |
| GPU temp (decode) | 70-75°C | 68-74°C |
| CPU temp | 71-73°C | 72-73°C |
| Throttling onset | ~100-150 tokens | — |
| Cold → warm drop | 30-40% | — |

### Thermal Throttling Impact

Sustained generation causes performance degradation:

```
Tokens 0-50:     152 tok/s (cold GPU, 66-70°C)
Tokens 100-200:  100-110 tok/s (warm GPU, 70-75°C)
Drop:            30-40%
```

**Why this happens**:
- GPU heats from 66°C → 75°C during inference
- macOS thermal management reduces GPU clocks
- Performance drops proportionally

**Mitigation**:
- Allow 15-30s cooldown between runs
- Use external cooling (laptop stands, fans)
- Run in cooler environments

## Optimizing for Energy Efficiency

### 1. Choose Q4_0 Quantization

Lower precision = less memory bandwidth = less power:

```bash
unc compile --model <model> --quant q4-0 -o ./model
```

**Energy impact**:
- F16: ~125 mJ/token
- Q8_0: ~100 mJ/token
- Q4_0: **74 mJ/token**

Q4_0 is **1.7x more energy-efficient** than F16.

### 2. Use JIT Mode for Development

JIT saves 1W CPU power vs AOT:

```bash
# JIT (default): 12.1W SoC
unc compile --model <model> --quant q4-0 -o ./model

# AOT: 13.5W SoC
unc compile --model <model> --quant q4-0 --binary -o ./model
```

Use JIT for development, AOT only for deployment.

### 3. Avoid Thermal Throttling

Throttling reduces efficiency:

```
Cold: 152 tok/s @ 11.2W = 74 mJ/token
Warm: 110 tok/s @ 11.2W = 102 mJ/token (38% worse)
```

**Solution**: Batch inference with cooldown periods

### 4. Close Background Applications

Other apps consume power and cause thermal interference:

```bash
# Check GPU usage
sudo powermetrics --samplers gpu_power -i 1000

# Close Chrome, Slack, etc. before long inference runs
```

### 5. Monitor Power Usage

Use macmon to track real-time power:

```bash
brew install vladkens/tap/macmon
macmon --interval 200
```

Target metrics:
- GPU power: 11-12W (decode)
- SoC power: 12-14W
- GPU utilization: 80-90%
