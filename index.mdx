## Overview

UNC (Universal Neural Compiler) transforms HuggingFace transformer models into highly optimized native inference binaries for Apple Silicon. By compiling models ahead-of-time or just-in-time, UNC eliminates Python runtime overhead and delivers near-hardware-limit performance.

### Performance Highlights

Running TinyLlama 1.1B on Apple M1 Pro (Q4_0 quantization):

- **152 tok/s** vs 113 tok/s (mlx-lm) — 1.35x faster
- **11.3W GPU power** vs 14.1W — 25% less power consumption
- **74 mJ/token** vs 125 mJ/token — 1.7x better energy efficiency
- **5.3B CPU instructions** vs 31.4B — 8.4x fewer instructions

## Supported Architectures

UNC supports the most popular transformer architectures:

- **LLaMA** (LLaMA, LLaMA-2, LLaMA-3)
- **Mistral** (Mistral, Mixtral)
- **Qwen** (Qwen, Qwen-2, Qwen-3)
- **Phi** (Phi-2, Phi-3)
- **Gemma** (Gemma, Gemma-2)

## Quick Example

```bash
# Compile a model from HuggingFace
unc compile --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 --quant q4-0 -o ./tinyllama

# Run inference
unc run ./tinyllama.unc --prompt "The history of" --max-tokens 200
```

## Architecture

UNC follows a traditional compiler architecture with a hardware-agnostic intermediate representation:

```
HuggingFace model → Frontend → IR Graph → Compiler → Metal/CUDA/ROCm/WASM → Native binary
```

The IR supports typed tensor operations including `BatchMatMul`, `QuantizedMatVec`, `RMSNorm`, `RoPE`, `SDPA`, `SwiGLU`, and more. Compiler passes perform optimizations like QKV fusion, elementwise fusion, memory planning, and kernel matching before lowering to target-specific code.
