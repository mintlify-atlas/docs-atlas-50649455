## Overview

Kernel fusion combines multiple operations into a single GPU dispatch, eliminating intermediate memory writes and reducing launch overhead. UNC's compiler identifies fusible patterns during the optimization pass and emits specialized fused kernels.

<Info>
**Performance gains from fusion:**
- **Memory traffic reduction**: 2-3× fewer DRAM accesses
- **Dispatch overhead**: Eliminate 50-70% of kernel launches
- **Latency hiding**: Improved arithmetic intensity
- **Energy efficiency**: 20-30% lower power consumption
</Info>

## Fusion Patterns

### 1. QKV Projection

**Fuses three matrix multiplications** that share the same input.

<Tabs>
  <Tab title="Pattern">
    **Unfused:**
    ```
    q = x @ W_q  →  GPU dispatch 1
    k = x @ W_k  →  GPU dispatch 2  
    v = x @ W_v  →  GPU dispatch 3
    ```
    
    **Fused:**
    ```
    (q, k, v) = qkv_projection(x, W_q, W_k, W_v)  →  Single dispatch
    ```
    
    **Memory savings:**
    - Unfused: Read `x` 3 times (3N reads)
    - Fused: Read `x` once (N reads)
    - **Reduction**: 67% fewer activation reads
  </Tab>
  <Tab title="Kernel (GEMV)">
    ```metal
    // dispatch: (ceil(M_q/64), 1, 1) x (256, 1, 1)
    // Fused QKV GEMV for decode path
    kernel void fused_qkv_gemv_f16(
        device half*             out_q  [[buffer(0)]],
        device const half*       A      [[buffer(1)]],
        device const half*       W_q    [[buffer(2)]],
        device half*             out_k  [[buffer(3)]],
        device const half*       W_k    [[buffer(4)]],
        device half*             out_v  [[buffer(5)]],
        device const half*       W_v    [[buffer(6)]],
        constant uint*           params [[buffer(7)]],
        uint  tgid_x  [[threadgroup_position_in_grid]],
        uint  simd_gid [[simdgroup_index_in_threadgroup]],
        uint  simd_lid [[thread_index_in_simdgroup]])
    {
        uint seq_len    = params[0];
        uint act_stride = params[1];   // 1 for decode
        uint K          = params[2];
        uint M_q        = params[3];
        uint M_kv       = params[4];
        
        uint m_base = tgid_x * GEMV_BM + simd_gid * GEMV_BR;
        
        for (uint n = 0u; n < seq_len; n++) {
            // --- Q projection (all threadgroups) ---
            {
                float acc[GEMV_BR] = {};
                uint k_vec = K / 4u;
                for (uint kv = simd_lid; kv < k_vec; kv += 32u) {
                    uint base_k = kv * 4u;
                    half4 a = *(device const half4*)(A + base_k * act_stride + n);
                    float a0 = float(a[0]), a1 = float(a[1]);
                    float a2 = float(a[2]), a3 = float(a[3]);
                    
                    for (uint r = 0u; r < GEMV_BR; r++) {
                        uint m = m_base + r;
                        if (m < M_q) {
                            half4 w = *(device const half4*)(W_q + m * K + base_k);
                            acc[r] += float(w[0]) * a0 + float(w[1]) * a1
                                    + float(w[2]) * a2 + float(w[3]) * a3;
                        }
                    }
                }
                for (uint r = 0u; r < GEMV_BR; r++) {
                    float total = simd_sum(acc[r]);
                    uint m = m_base + r;
                    if (simd_lid == 0u && m < M_q) {
                        out_q[m * act_stride + n] = half(total);
                    }
                }
            }
            
            // --- K projection (only if this threadgroup covers M_kv) ---
            if (m_base < M_kv) {
                float acc[GEMV_BR] = {};
                uint k_vec = K / 4u;
                for (uint kv = simd_lid; kv < k_vec; kv += 32u) {
                    uint base_k = kv * 4u;
                    half4 a = *(device const half4*)(A + base_k * act_stride + n);
                    float a0 = float(a[0]), a1 = float(a[1]);
                    float a2 = float(a[2]), a3 = float(a[3]);
                    
                    for (uint r = 0u; r < GEMV_BR; r++) {
                        uint m = m_base + r;
                        if (m < M_kv) {
                            half4 w = *(device const half4*)(W_k + m * K + base_k);
                            acc[r] += float(w[0]) * a0 + float(w[1]) * a1
                                    + float(w[2]) * a2 + float(w[3]) * a3;
                        }
                    }
                }
                for (uint r = 0u; r < GEMV_BR; r++) {
                    float total = simd_sum(acc[r]);
                    uint m = m_base + r;
                    if (simd_lid == 0u && m < M_kv) {
                        out_k[m * act_stride + n] = half(total);
                    }
                }
            }
            
            // --- V projection (same as K) ---
            if (m_base < M_kv) {
                // ... similar structure
            }
        }
    }
    ```
  </Tab>

### 2. SwiGLU (Gated FFN)

**Fuses three operations**: gate projection, up projection, and gated activation.

<Tabs>
  <Tab title="Pattern">
    **Unfused:**
    ```python
    gate = x @ W_gate      # Linear 1
    up   = x @ W_up        # Linear 2
    gate = silu(gate)      # Activation
    out  = gate * up       # Element-wise multiply
    ```
    4 dispatches, 3 intermediate buffers
    
    **Fused:**
    ```python
    out = swiglu(x, W_gate, W_up)  # Single dispatch, no intermediate buffers
    ```
    
    **Formula:**
    ```
    SwiGLU(x) = SiLU(x @ W_gate) ⊙ (x @ W_up)
              = (x @ W_gate) / (1 + exp(-(x @ W_gate))) ⊙ (x @ W_up)
    ```
  </Tab>
  <Tab title="Kernel">
    ```metal
    // Simplified SwiGLU kernel (decode path)
    kernel void swiglu_gemv_f16(
        device half*             out    [[buffer(0)]],
        device const half*       x      [[buffer(1)]],
        device const half*       W_gate [[buffer(2)]],
        device const half*       W_up   [[buffer(3)]],
        constant uint*           params [[buffer(4)]],
        uint  tgid_x  [[threadgroup_position_in_grid]],
        uint  simd_gid [[simdgroup_index_in_threadgroup]],
        uint  simd_lid [[thread_index_in_simdgroup]])
    {
        uint K = params[2];  // input hidden size
        uint M = params[3];  // output hidden size (ffn_dim)
        
        uint m_base = tgid_x * GEMV_BM + simd_gid * GEMV_BR;
        
        for (uint r = 0u; r < GEMV_BR; r++) {
            uint m = m_base + r;
            if (m >= M) continue;
            
            // Compute gate and up projections in parallel
            float gate_acc = 0.0f;
            float up_acc = 0.0f;
            
            uint k_vec = K / 4u;
            for (uint kv = simd_lid; kv < k_vec; kv += 32u) {
                uint base_k = kv * 4u;
                half4 x_vec = *(device const half4*)(x + base_k);
                float x0 = float(x_vec[0]), x1 = float(x_vec[1]);
                float x2 = float(x_vec[2]), x3 = float(x_vec[3]);
                
                // Gate projection
                half4 w_gate = *(device const half4*)(W_gate + m * K + base_k);
                gate_acc += float(w_gate[0]) * x0 + float(w_gate[1]) * x1
                          + float(w_gate[2]) * x2 + float(w_gate[3]) * x3;
                
                // Up projection
                half4 w_up = *(device const half4*)(W_up + m * K + base_k);
                up_acc += float(w_up[0]) * x0 + float(w_up[1]) * x1
                        + float(w_up[2]) * x2 + float(w_up[3]) * x3;
            }
            
            // Reduce across simdgroup
            gate_acc = simd_sum(gate_acc);
            up_acc = simd_sum(up_acc);
            
            // Apply SiLU to gate and multiply
            if (simd_lid == 0u) {
                float gate_silu = gate_acc / (1.0f + exp(-gate_acc));
                out[m] = half(gate_silu * up_acc);
            }
        }
    }
    ```
  </Tab>
  <Tab title="Performance">
    **Prefill (seq_len=128) benchmarks** (Llama-2-7B FFN, M1 Pro):
    
    | Approach | Latency | Memory | Arithmetic Intensity |
    |----------|---------|--------|---------------------|
    | Unfused | 2.8ms | 100 MB | 1.2 FLOP/byte |
    | Fused | 1.9ms | 68 MB | 1.8 FLOP/byte |
    | **Speedup** | **1.47×** | **32% less** | **1.5× higher** |
    
    **Why it works:**
    - Gate and up projections read same `x` input
    - SiLU and multiply are cheap (fused into register operations)
    - No materialization of intermediate `gate` and `up` tensors
  </Tab>
</Tabs>

### 3. Add + RMSNorm

**Fuses residual addition with normalization** (common in transformer blocks).

<Tabs>
  <Tab title="Pattern">
    **Unfused:**
    ```python
    residual = x + delta        # Element-wise add
    out = rms_norm(residual)    # Normalization
    ```
    2 dispatches, 1 intermediate buffer (`residual`)
    
    **Fused:**
    ```python
    (out, residual) = add_rms_norm(x, delta, weight)  # Single dispatch
    ```
    Returns both normalized output AND residual for next layer.
  </Tab>
  <Tab title="Kernel">
    ```metal
    // dispatch: (seq_len, 1, 1) x (256, 1, 1)
    kernel void add_rms_norm_f16(
        device half*             out_norm     [[buffer(0)]],
        device const half*       x            [[buffer(1)]],
        device const half*       delta        [[buffer(2)]],
        device const half*       weight       [[buffer(3)]],
        constant uint*           params       [[buffer(4)]],
        device half*             out_residual [[buffer(5)]],
        uint tgid_x [[threadgroup_position_in_grid]],
        uint lid_x  [[thread_position_in_threadgroup]],
        threadgroup float*       shared       [[threadgroup(0)]])
    {
        uint seq_len    = params[0];
        uint act_stride = params[1];
        uint h          = params[2];
        float eps       = as_type<float>(params[3]);
        
        uint si = tgid_x;
        if (si >= seq_len) return;
        
        // Pass 1: Compute residual = x + delta, accumulate sum of squares
        float partial_sq = 0.0f;
        for (uint hi = lid_x; hi < h; hi += 256u) {
            uint idx = hi * act_stride + si;
            float sum = float(x[idx]) + float(delta[idx]);
            out_residual[idx] = half(sum);  // Write residual for next layer
            partial_sq += sum * sum;
        }
        shared[lid_x] = partial_sq;
        threadgroup_barrier(mem_flags::mem_threadgroup);
        
        // Tree reduction for RMS
        for (uint stride = 128u; stride > 0u; stride >>= 1u) {
            if (lid_x < stride) {
                shared[lid_x] += shared[lid_x + stride];
            }
            threadgroup_barrier(mem_flags::mem_threadgroup);
        }
        
        float scale = rsqrt(shared[0] / float(h) + eps);
        threadgroup_barrier(mem_flags::mem_threadgroup);
        
        // Pass 2: Read residual back and apply normalization
        for (uint hi = lid_x; hi < h; hi += 256u) {
            uint idx = hi * act_stride + si;
            float v = float(out_residual[idx]);
            out_norm[idx] = half(v * scale * float(weight[hi]));
        }
    }
    ```
  </Tab>
  <Tab title="Performance">
    **Decode (seq_len=1) benchmarks** (Llama-2-7B, hidden=4096, M1 Pro):
    
    | Approach | Latency | Memory Writes | Dispatches |
    |----------|---------|---------------|------------|
    | Unfused | 0.14ms | 2 × 8 KB | 2 |
    | Fused | 0.09ms | 2 × 8 KB | 1 |
    | **Speedup** | **1.56×** | Same | **2× fewer** |
    
    **Notes:**
    - Fused version still writes `residual` (needed for next block)
    - Speedup comes from eliminating dispatch overhead and reusing loaded data
    - Prefill (seq_len=128): 1.8× speedup (32 KB → 16 KB traffic)
  </Tab>
</Tabs>

### 4. RMSNorm + Linear (Decode)

**Fuses normalization with following matrix multiply** (decode path only).

<Warning>
This fusion is **only beneficial for decode** (seq_len=1) where GEMV is memory-bound. For prefill, the overhead of computing normalization within the GEMM kernel hurts performance.
</Warning>

<Tabs>
  <Tab title="Pattern">
    **Unfused:**
    ```python
    x_norm = rms_norm(x, weight_norm)  # Normalization
    out = x_norm @ W                   # Linear (GEMV)
    ```
    
    **Fused:**
    ```python
    out = rms_norm_linear(x, weight_norm, W)  # Single dispatch
    ```
    
    **Why it helps (decode):**
    - Eliminate write of `x_norm` (8 KB for hidden=4096)
    - Normalize on-the-fly as elements are read
    - Weight streaming (W) dominates latency anyway
  </Tab>
  <Tab title="Concept">
    ```metal
    kernel void rms_norm_linear_gemv_f16(...) {
        // Phase 1: Compute RMS scale (same as standalone RMSNorm)
        float rms = 0.0f;
        for (uint k = 0; k < K; k++) {
            float v = float(x[k]);
            rms += v * v;
        }
        rms = rsqrt(rms / float(K) + eps);
        
        // Phase 2: Normalize x and immediately accumulate into GEMV
        for (uint m = m_base; m < m_base + GEMV_BR; m++) {
            float acc = 0.0f;
            for (uint k = 0; k < K; k++) {
                // Normalize x[k] on-the-fly
                float x_norm = float(x[k]) * rms * float(weight_norm[k]);
                acc += x_norm * float(W[m * K + k]);
            }
            out[m] = half(simd_sum(acc));
        }
    }
    ```
    
    **Eliminated:** Intermediate `x_norm` buffer (4-8 KB per token)
  </Tab>
  <Tab title="Performance">
    **Decode (seq_len=1) benchmarks** (Qwen3-4B, M1 Pro):
    
    | Approach | Latency | Memory Traffic | Dispatches |
    |----------|---------|----------------|------------|
    | Unfused | 0.38ms | 16 MB | 2 |
    | Fused | 0.31ms | 15.98 MB | 1 |
    | **Speedup** | **1.23×** | **0.1% less** | **2× fewer** |
    
    **Marginal gains** because:
    - Weight reads (W) dominate memory traffic
    - Savings from eliminating `x_norm` write are small relative to total
    - Main benefit is reduced dispatch overhead
  </Tab>
</Tabs>

### 5. RoPE Q+K Fusion

**Applies rotary embeddings to Q and K in a single dispatch**.

<Tabs>
  <Tab title="Pattern">
    **Unfused:**
    ```python
    q_rope = rope(q, position)  # RoPE for queries
    k_rope = rope(k, position)  # RoPE for keys
    ```
    
    **Fused:**
    ```python
    (q_rope, k_rope) = rope_qk(q, k, position, rope_table)
    ```
  </Tab>
  <Tab title="Kernel">
    ```metal
    // dispatch: (seq_len, 1, 1) x (256, 1, 1)
    kernel void rope_qk_f16(
        device half*             out_q  [[buffer(0)]],
        device const half*       q      [[buffer(1)]],
        device half*             out_k  [[buffer(2)]],
        device const half*       k      [[buffer(3)]],
        constant uint*           params [[buffer(4)]],
        device const half*       rope_table [[buffer(5)]],
        uint tgid_x [[threadgroup_position_in_grid]],
        uint lid_x  [[thread_position_in_threadgroup]])
    {
        uint seq_len      = params[0];
        uint act_stride   = params[1];
        uint rope_dim_q   = params[2];  // num_q_heads * head_dim
        uint head_dim     = params[3];
        uint total_seq    = params[6];
        uint rope_dim_k   = params[7];  // num_kv_heads * head_dim (GQA)
        
        uint si = tgid_x;
        if (si >= seq_len) return;
        
        uint abs_pos = total_seq - seq_len + si;
        uint half_dim = head_dim / 2u;
        
        // Process Q pairs
        uint n_pairs_q = rope_dim_q / 2u;
        uint chunk_q = (n_pairs_q + 255u) / 256u;
        uint p_start = lid_x * chunk_q;
        uint p_end   = min(p_start + chunk_q, n_pairs_q);
        
        for (uint pi = p_start; pi < p_end; pi++) {
            uint head_idx = pi / half_dim;
            uint pair_pos = pi % half_dim;
            uint hi0 = head_idx * head_dim + pair_pos;
            uint hi1 = head_idx * head_dim + pair_pos + half_dim;
            
            // Precomputed cos/sin from table
            uint tidx = abs_pos * half_dim + pair_pos;
            float cos_t = float(rope_table[tidx * 2u]);
            float sin_t = float(rope_table[tidx * 2u + 1u]);
            
            // 2D rotation for Q
            float q0 = float(q[hi0 * act_stride + si]);
            float q1 = float(q[hi1 * act_stride + si]);
            out_q[hi0 * act_stride + si] = half(q0 * cos_t - q1 * sin_t);
            out_q[hi1 * act_stride + si] = half(q0 * sin_t + q1 * cos_t);
        }
        
        // Process K pairs (same structure, different dimensions for GQA)
        uint n_pairs_k = rope_dim_k / 2u;
        uint chunk_k = (n_pairs_k + 255u) / 256u;
        p_start = lid_x * chunk_k;
        p_end   = min(p_start + chunk_k, n_pairs_k);
        
        for (uint pi = p_start; pi < p_end; pi++) {
            // ... same rotation logic for K
        }
    }
    ```
  </Tab>
  <Tab title="Performance">
    **Prefill (seq_len=128) benchmarks** (Qwen3-4B, 32 Q heads, 8 KV heads, M1 Pro):
    
    | Approach | Latency | Dispatches |
    |----------|---------|------------|
    | Unfused (2×) | 0.18ms | 2 |
    | Fused | 0.11ms | 1 |
    | **Speedup** | **1.64×** | **2× fewer** |
    
    **Speedup factors:**
    - Shared position computation (abs_pos, rope_table lookup)
    - Eliminated dispatch overhead
    - Better instruction cache utilization

### 6. Ultra Fusion: SDPA + RoPE + KV Append (Decode)

**The ultimate decode fusion**: Combines RoPE, KV cache write, and attention in a **single dispatch**.

<Tabs>
  <Tab title="Pattern">
    **Unfused (5 dispatches):**
    ```python
    q_rope = rope(q, position)          # 1: RoPE Q
    k_rope = rope(k, position)          # 2: RoPE K
    k_cache[:, pos] = k_rope            # 3: Append K to cache
    v_cache[:, pos] = v                 # 4: Append V to cache
    out = sdpa(q_rope, k_cache, v_cache) # 5: Attention
    ```
    
    **Ultra-fused (1 dispatch):**
    ```python
    out = sdpa_rope_kv_decode(q, k, v, k_cache, v_cache, position, rope_table)
    ```
  </Tab>
  <Tab title="Kernel (Excerpt)">
    ```metal
    // dispatch: (num_q_heads, 1, 1) x (32, 32, 1)  [32 simdgroups × 32 threads]
    kernel void sdpa_rope_kv_decode_f16(
        device half*             out         [[buffer(0)]],
        device const half*       raw_q       [[buffer(1)]],
        device const half*       raw_k       [[buffer(2)]],
        device const half*       raw_v       [[buffer(3)]],
        device half*             kv_k_cache  [[buffer(4)]],
        device half*             kv_v_cache  [[buffer(5)]],
        device const half*       rope_table  [[buffer(6)]],
        constant uint*           params      [[buffer(7)]],
        uint  tgid_x  [[threadgroup_position_in_grid]],
        uint  simd_gid [[simdgroup_index_in_threadgroup]],
        uint  simd_lid [[thread_index_in_simdgroup]])
    {
        constexpr int BN = 32;       // simdgroups
        constexpr int BD = 32;       // threads per simdgroup
        constexpr int HEAD_DIM = 64; // compile-time head dimension
        
        uint q_head_idx  = tgid_x;
        uint kv_head_idx = q_head_idx / gqa_factor;
        int  abs_pos     = N - 1;  // position of current token
        
        // RoPE pairing: thread t pairs with thread t+16
        uint pair_thread = simd_lid < 16u ? simd_lid + 16u : simd_lid - 16u;
        
        // ---- Phase 1: Apply RoPE to Q inline (all threads) ----
        float q[2];  // HEAD_DIM/BD = 64/32 = 2 per thread
        for (int i = 0; i < 2; i++) {
            q[i] = float(raw_q[q_head_idx * HEAD_DIM + simd_lid * 2 + i]);
        }
        for (int i = 0; i < 2; i++) {
            float q_pair = simd_shuffle(q[i], pair_thread);
            uint pos_in_head = simd_lid * 2 + i;
            uint pair_pos = pos_in_head < 32u ? pos_in_head : pos_in_head - 32u;
            uint tidx = abs_pos * 32u + pair_pos;
            float cos_t = float(rope_table[tidx * 2u]);
            float sin_t = float(rope_table[tidx * 2u + 1u]);
            if (pos_in_head < 32u) {
                q[i] = q[i] * cos_t - q_pair * sin_t;
            } else {
                q[i] = q_pair * sin_t + q[i] * cos_t;
            }
            q[i] *= scale;
        }
        
        // ---- Phase 2: RoPE K + store K/V in threadgroup memory ----
        threadgroup half tg_k[HEAD_DIM];
        threadgroup half tg_v[HEAD_DIM];
        
        if (simd_gid == 0) {
            // Load, RoPE K, store to TG memory
            // Load V, store to TG memory
        }
        threadgroup_barrier(mem_flags::mem_threadgroup);
        
        // ---- Phase 3: SDPA over cached positions 0..N-2 ----
        float o[2] = {};
        float max_score = -__FLT_MAX__;
        float sum_exp_score = 0.0f;
        
        for (int i = simd_gid; i < N - 1; i += BN) {
            // Load K from cache, compute Q·K, online softmax, accumulate V
        }
        
        // Attend to CURRENT position (N-1) using threadgroup memory K/V
        // (no device memory fence needed)
        
        // ---- Phase 4: Reduce across simdgroups ----
        // Write attention output
        
        // ---- Phase 5: Write K/V to cache AFTER attention ----
        // Serial dispatch ordering ensures future tokens see these writes
        if (simd_gid == 0) {
            for (int i = 0; i < 2; i++) {
                uint pos_in_head = simd_lid * 2 + i;
                uint cache_idx = kv_head_idx * max_seq * HEAD_DIM + abs_pos * HEAD_DIM + pos_in_head;
                kv_k_cache[cache_idx] = tg_k[pos_in_head];
                kv_v_cache[cache_idx] = tg_v[pos_in_head];
            }
        }
    }
    ```
  </Tab>
  <Tab title="Performance">
    **Decode (seq_len=1, context=1024) benchmarks** (TinyLlama 1.1B, M1 Pro):
    
    | Approach | Latency | Dispatches | Memory Fences |
    |----------|---------|------------|---------------|
    | Unfused | 0.62ms | 5 | 3 |
    | Fused | 0.48ms | 1 | 0 |
    | **Speedup** | **1.29×** | **5× fewer** | **0 fences** |
    
    **Key innovations:**
    
    1. **Threadgroup memory for current token**:
       - K/V for position N-1 stored in SRAM (32 KB)
       - Attention reads from TG memory (no device memory fence)
       - Writes to device memory AFTER attention completes
    
    2. **Inline RoPE via simd_shuffle**:
       - Thread t holds positions [2t, 2t+1]
       - Thread t+16 holds paired positions [2t+64, 2t+65]
       - `simd_shuffle(q[i], pair_thread)` exchanges values
       - No explicit loops or barriers
    
    3. **Serial dispatch ordering**:
       - UNC compiler ensures serial execution across layers
       - KV cache writes visible to next layer without explicit fence
       - Eliminates expensive `mem_flags::mem_device` barriers
  </Tab>
</Tabs>

## Fusion Decision Logic

The UNC compiler applies fusion based on:

### 1. Pattern Recognition

```rust
// src/compile/fusion_pass.rs
match (op_sequence) {
    [Op::MatVec, Op::MatVec, Op::MatVec] if same_input => FusionPattern::QKVProjection,
    [Op::MatVec, Op::MatVec, Op::SiLU, Op::Mul] => FusionPattern::SwiGLU,
    [Op::Add, Op::RMSNorm] => FusionPattern::AddRMSNorm,
    [Op::RoPE, Op::RoPE] if sequential => FusionPattern::RoPEQK,
    _ => NoFusion,
}
```

### 2. Profitability Analysis

**Fusion is profitable when:**

```
savings = (eliminated_writes + eliminated_reads) * bandwidth_cost
        + eliminated_dispatches * dispatch_overhead
        
cost = increased_register_pressure * spill_cost
     + increased_code_size * icache_miss_cost
     
if savings > cost + fusion_threshold:
    apply_fusion()
```

**Typical thresholds:**
- Prefill: Savings > 10% memory traffic
- Decode: Savings > 5% (dispatch overhead dominates)

### 3. Memory Constraints

```rust
if threadgroup_memory_required > target.max_threadgroup_memory:
    skip_fusion()  // Exceeds 32 KB (Apple7) or 64 KB (Apple9+)
    
if register_file_pressure > 0.85:
    skip_fusion()  // Risk of register spills to memory
```

## Fusion Limitations

<Warning>
**When fusion is NOT applied:**

1. **Multiple consumers**: If intermediate result is used by >1 op
   ```python
   x_norm = rms_norm(x)
   q = x_norm @ W_q  # Consumer 1
   k = x_norm @ W_k  # Consumer 2
   # Cannot fuse RMSNorm into Q or K (both need x_norm)
   ```

2. **Cross-layer fusion**: UNC only fuses within a layer
   ```python
   # Layer i: out_i = ...
   # Layer i+1: out_{i+1} = f(out_i)
   # No cross-layer fusion (KV cache dependencies)
   ```

3. **Dynamic shapes**: Some fusions require compile-time dimensions
   ```python
   # OK: head_dim known at compile time (64 or 128)
   # Not OK: seq_len unknown (prefill vs decode)
   ```

4. **Precision constraints**: Mixed-precision fusion is complex
   ```python
   # FP16 + Q4_0 fusion requires careful dequantization
   # Currently not fused (future work)
   ```
</Warning>
