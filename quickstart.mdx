Get started with UNC by compiling TinyLlama and running your first inference.

## Prerequisites

Before you begin, make sure you have:

- macOS with Apple Silicon (M1, M2, M3, or M4)
- UNC installed (see [Installation](/installation))
- At least 8GB of available disk space

## Compile Your First Model

<Steps>
  <Step title="Compile TinyLlama with Q4_0 quantization">
    Start with a small model that compiles quickly:

    ```bash
    unc compile \
      --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 \
      --quant q4-0 \
      -o ./tinyllama
    ```

    This will:
    - Download the model from HuggingFace (auto-cached)
    - Parse the model architecture (config.json)
    - Build the computation graph
    - Apply optimization passes (QKV fusion, memory planning, etc.)
    - Generate Metal kernels
    - Produce `tinyllama.unc` bundle

    **Expected output:**
    ```
    unc: compiling TinyLlama/TinyLlama-1.1B-Chat-v1.0 → ./tinyllama.unc
      arch=LLaMA, layers=22, heads=32 (kv=4), hidden=2048, ffn=5632
      graph: 156 nodes, quant: q4_0
      registry: 47 kernels for metal_m1
      compile: 89 nodes, 23 kernels, activation=128 MB, kv=512 MB
    Done → ./tinyllama.unc
    ```

    <Note>
      First compilation may take 2-5 minutes. Subsequent compilations are faster due to HuggingFace caching.
    </Note>
  </Step>

  <Step title="Run inference">
    Run the compiled model:

    ```bash
    unc run ./tinyllama.unc --prompt "The history of" --max-tokens 200
    ```

    You should see output like:
    ```
    The history of the United States is a complex and fascinating story...
    [152.0 tok/s, 200 tokens in 1.32s]
    ```

    <Tip>
      On M1 Pro with Q4_0 quantization, expect around 150 tok/s for TinyLlama.
    </Tip>
  </Step>

  <Step title="Try different prompts">
    Experiment with different inputs:

    ```bash
    # Creative writing
    unc run ./tinyllama.unc --prompt "Once upon a time" --max-tokens 100

    # Technical explanation
    unc run ./tinyllama.unc --prompt "Explain quantum computing:" --max-tokens 150

    # Multi-line prompt
    unc run ./tinyllama.unc --prompt "Question: What is machine learning?
    Answer:" --max-tokens 100
    ```
  </Step>
</Steps>

## Compare Quantization Options

Try different quantization levels to see the speed vs quality tradeoff:

<CodeGroup>

```bash Q4_0 (Fastest)
unc compile --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 --quant q4-0 -o ./tinyllama-q4
# Size: 600 MB, Speed: ~152 tok/s
```

```bash Q8_0 (Balanced)
unc compile --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 --quant q8-0 -o ./tinyllama-q8
# Size: 1.1 GB, Speed: ~76 tok/s
```

```bash F16 (Best Quality)
unc compile --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 --quant f16 -o ./tinyllama-f16
# Size: 2.2 GB, Speed: ~48 tok/s
```

</CodeGroup>

| Quantization | Size | Speed | Quality | Use Case |
|--------------|------|-------|---------|----------|
| **Q4_0** | 600 MB | 152 tok/s | Good | Production, fast inference |
| **Q8_0** | 1.1 GB | 76 tok/s | Better | Quality-sensitive apps |
| **F16** | 2.2 GB | 48 tok/s | Best | Benchmarking, reference |

## Try Larger Models

Once comfortable with TinyLlama, try larger models:

<Tabs>
  <Tab title="Qwen3-4B">
    ```bash
    unc compile --model Qwen/Qwen3-4B --quant q4-0 -o ./qwen3
    unc run ./qwen3.unc --prompt "Write a haiku about compilers:" --max-tokens 50
    ```
    Expected performance: ~39 tok/s on M1 Pro
  </Tab>

  <Tab title="Phi-3-Mini">
    ```bash
    unc compile --model microsoft/Phi-3-mini-4k-instruct --quant q4-0 -o ./phi3
    unc run ./phi3.unc --prompt "Explain tensors:" --max-tokens 100
    ```
  </Tab>

  <Tab title="Mistral-7B">
    ```bash
    unc compile --model mistralai/Mistral-7B-v0.1 --quant q4-0 -o ./mistral
    unc run ./mistral.unc --prompt "The future of AI is" --max-tokens 150
    ```
    Requires 16GB+ RAM for compilation
  </Tab>
</Tabs>

## Create a Standalone Binary

For deployment, compile to a standalone binary with embedded weights:

```bash
unc compile \
  --model TinyLlama/TinyLlama-1.1B-Chat-v1.0 \
  --quant q4-0 \
  --binary \
  -o ./tinyllama-standalone

# Run the binary directly
./tinyllama-standalone "The meaning of life is" --max-tokens 100
```

<Note>
  AOT binaries are self-contained (600MB-2GB) but start instantly (less than 100ms) and have zero dependencies.
</Note>

## Inspect Compiled Models

View details about a compiled model:

```bash
# Basic info
unc inspect ./tinyllama.unc

# With IR dump
unc inspect ./tinyllama.unc --dump-ir
```

**Example output:**
```
=== ./tinyllama.unc ===
model:  TinyLlama/TinyLlama-1.1B-Chat-v1.0
arch:   LLaMA
target: METAL
nodes:  89
stats:  nodes_after=89 unique_kernels=23 fusions_applied=18
```

## Common Issues

<AccordionGroup>
  <Accordion title="Model download is slow">
    HuggingFace downloads can take time on first run. Models are cached in `~/.cache/huggingface/hub/` for future use.
  </Accordion>

  <Accordion title="Out of memory during compilation">
    Try:
    - Closing other apps
    - Using a smaller model first
    - Reducing `--max-seq-len` (default: 4096)
  </Accordion>

  <Accordion title="Slower than expected">
    - First run includes JIT compilation overhead (~1s)
    - Performance degrades when GPU is hot (thermal throttling)
    - Check Activity Monitor for competing GPU workloads
  </Accordion>
</AccordionGroup>
